Ready steady go - interacting with the dom

Published on 2013-7-23

The very first thing I usually want to do with a project, is to do something with the dom.

  • Look ma, no frameworks
  • Grunt+browserify+npm+application=success

Okay, so first things first - how do I wait for the dom to be ready so I can do stuff with it?

Don't

<html>
  <body>
    <div id="content"></div>
    <script src="type="text/javascript"></script>
  </body>

Is one way of achieving this, stick the JS at the end of the file and run stuff from here.

Actually, you could read/write about this subject until the cows come home and enough people have, but I just wanted to make the point that you don't need to do:

$(function() {
  // My Application
})

Just for the sake of it.

Avoiding jQuery

I don't really use jQuery any more, it's a bit too swiss army knife for me and while it might "protect you from the common problems between browsers", most of the basic DOM manipulations aren't really that exciting between browers any more.

However, waiting for the DOM to be ready is still potentially complicated and managing this stuff ourselves doesn't seem very worthwhile.

NPM to the rescue

npm install domready --save-dev
vim client/app.js
var domready = require('domready')

domready(function() {
  // Application code goes here
})
